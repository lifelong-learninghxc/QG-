不同通信网络拓扑下网联自动驾驶车辆的反馈式编队控制

**主要任务**：针对不同通信网络拓扑下的网联自动驾驶车辆（CAVs），提出一种基于反馈的编队控制协议，
具体来说是（采用代数图论描述车辆通信拓扑，设计同时考虑纵向与横向间距的反馈式编队控制协议，基于李雅普诺夫方法与拉萨尔不变性原理分析系统稳定性与一致性，并探究不同拓扑对编队控制收敛性与鲁棒性的影响。）

**关键词**：网联自动驾驶车辆；编队控制；通信网络拓扑；一致性


**创新点**：在自动驾驶中搭建通信技术，可以更好的了解实时的交通情况，更高效的避障和决策。同时保证编队所有车辆以相同速度行驶，保持队伍的总体队形，这就需要本文所提出的创新点来强化。


**提出多种队形控制方法**：**领航 - 跟随**、**虚拟领航**等
1.领航—跟随方法采用所有车辆收敛向领航者的方式，可以逐渐跟踪逼近领航者。
2.虚拟领航方法通过引入刚性虚拟结构的粒子来进行虚拟化领航，有助于避免领航 - 跟随方法固有的扰动问题。

对公式的理解：这是两个微分方程，第二个式子是通过协调跟踪车的加速度，不断的适配领航车的加速度，使他们都向领航车靠拢。
$$-\sum_{j=1}^{n} a_{ij} [(x_i(t) - x_j(t) - r_{ij}) + \beta(v_i(t) - v_j(t))]$$
这一项是协调跟踪车之间的运行， **$a_{ij}$** 表示邻接权重，$r_{ij}$ 控制了每辆跟踪车之间的预设间距，**$\beta(v_i - v_j)$ 作为速度偏差项。旨在让两车的速度趋于一致。
$$- k_i [(x_i(t) - x_L(t) - r_i) + \gamma(v_i(t) - v_L(t))]$$
 $k_i$ 表示连接权重。$r_i$ 是车辆 $i$ 与领航车之间的期望间距。$\gamma(v_i - v_L)$**：相对于领航车的速度偏差，$\gamma$ 是速度反馈增益。

这样一个算法实现了跟踪车之间保持稳定状态运行，同时也可以逐渐向领航车的运动状态收敛，包含一种反馈调节机制，可以实时通过外界状态来对自身的运动进行规范。

$\tilde{v}_i(t) = v_i(t) - \boldsymbol{1}v_L(t)$

这个公式的含义是：**第 $i$ 辆车当前的速度与领航者（Leader）速度之间的差值。其中包含了水平与竖直两个方向。

$\tilde{x}_i(t) = x_i(t) - \boldsymbol{1} \int_{0}^{t} v_L(\tau) \text{d}\tau - r_i$

这个公式定义了**第 $i$ 辆车是否处于其“理想位置”**。
*   **$x_i(t)$**：跟随车 $i$ 的当前实际位置。
*   **$\int_{0}^{t} v_L(\tau) \text{d}\tau$**：这一项是领航者速度随时间的积分，本质上代表了**领航者从起始点开始走过的总路程**。
*   **$r_i$**：这是**期望间距向量**。它包含了这辆车相对于领航者应该保持的纵向距离和横向距离。

![[Pasted image 20260322215903.png]]
$\dot{\tilde{x}}_i(t) = \tilde{v}_i(t)$ 它表示：**位置误差的变化率，就是速度误差。**
$$-\sum_{j=1}^{n} a_{ij} [(\tilde{x}_i(t) - \tilde{x}_j(t)) + \beta(\tilde{v}_i(t) - \tilde{v}_j(t))]$$
 消除车辆之间的相对间距误差和相对速度差。
    $$- k_i (\tilde{x}_i(t) + \gamma \tilde{v}_i(t))$$
 领航车拉动车辆 $i$ 向着 $\tilde{x}_i = 0$ 和 $\tilde{v}_i = 0$（即理想的绝对位置和速度）收敛。只有能接收到领航者信号的车（$k_i > 0$），这一项才会起作用。


引用理论1：![[Pasted image 20260322221150.png]]
在所有车都必须能直接或间接地受到领航者的影响同时时间 $t$ 趋于无穷大时：位置误差 $\tilde{x}_i \to 0$ 速度误差 $\tilde{v}_i \to 0$。

证明是构造一个**李雅普诺夫函数 $V(t)$**。即系统的总误差能量。
$$V(t) = V_1(t) + V_2(t) + V_3(t)$$
*   **$V_1$ (速度能量)**：基于速度误差的平方。如果大家速度不统一，这个能量就很高。
*   **$V_2$ (邻居间位置势能)**：两两车辆之间如果没有排整齐，就像弹簧被拉开一样，产生“势能”。
*   **$V_3$ (与领航者间位置势能)**：跟随车如果没有跟准领航者，产生的能量。

    如果能证明这个“总能量” $V(t)$ 随时间一直在减少（导数 $\dot{V} \le 0$），且最终会耗尽，那么误差就一定会消失。

 公式 (6) 到 (8)通过对 $V$ 求导，得到了系统能量的变化率 $\dot{V}$。
***公式 (8)** 利用了网络拓扑是“无向图”（即 $i$ 看到 $j$，$j$ 也能看到 $i$）的对称性进行简化。
***关键结论**：$\dot{V} = -\dots \le 0$。
 这个负号代表系统的总偏差能量在不断损耗。系统是**稳定**的。

公式 (9) 到 (11)证明它最后一定会停在“零误差”的状态。
利用了 **LaSalle 不变性原理**：

公式 (9)**：如果能量不再减少了（$\dot{V} = 0$），意味着所有车的速度必须已经相等且等于领航者速度（$\tilde{v}_i = 0$）。

**公式 (10) & (11)**：如果速度已经同步了，但位置还没排好（即 $\tilde{x}_i \neq 0$），那么车辆之间依然会有相互的作用力（加速度）。
 
通过公式 (11) 证明，只有当位置误差 $\tilde{x}_i$ 也全部为 0 时，系统才能真正“静止”在稳定状态。

局部断连（图 7, 8, 9）
这个实验是为了测试系统的“鲁棒性”（健壮性）：
*   **关键发现**：由于车辆 $i$ 失去了所有联系（如 Fig. 7 蓝线所示），它**掉队了**。它无法接收信息，所以只能按原样跑或乱跑，没法加入编队。
*   **好的表现**：虽然 $i$ 掉了，但剩下的车辆（$i+1$ 和 $i+2$）因为互相还能通信，它们**依然成功形成了两个车的小编队**。
*   **结论**：算法在部分节点失效时，剩余的连通部分依然能保持稳定。

*   **Case I（全连通网络，3辆跟随车）**：
    *   纵向收敛需要 **7.9秒**。
    *   横向收敛需要 **9.2秒**。
*   **Case II（局部断连，实际上只有2辆跟随车参与有效编队）**：
    *   纵向收敛需要 **6.5秒**。
    *   横向收敛需要 **7.4秒**。
**系统复杂度降低**：在 Case I 中，有 3 辆跟随车相互制约、相互拉扯，达成共识需要的协调次数更多。
**“掉队”带来的简化**：在 Case II 中，由于第 $i$ 辆车已经失联并掉队，剩下的只有 2 辆车在相互配合。**小组规模越小，成员之间达成步调一致的速度就越快。**
**约束减少**：车与车之间的通信链路变少了，算法需要处理的耦合信息量也随之减少，从而加快了剩余车队达到稳定状态的速度。
