## 1. 贝叶斯公式推导与“朴素”假设

贝叶斯分类器的核心思想是**逆向概率推理**：当我们看到某些特征（结果）时，推断出它属于某个类别（原因）的概率。

### 1.1 贝叶斯公式的严谨推导

根据条件概率的定义，事件 B 发生的前提下事件 A 发生的概率为：

$$P(A|B) = \frac{P(A \cap B)}{P(B)}$$

同理，事件 A 发生的前提下事件 B 发生的概率为：

$$P(B|A) = \frac{P(A \cap B)}{P(A)}$$

将两式中的联合概率$P(A \cap B)$提取并等价：

$$P(A|B)P(B) = P(B|A)P(A)$$

将$P(B)$移项，我们就得到了极其优雅的**贝叶斯定理 （贝叶斯定理）：**

$$P(A|B) = \frac{P(B|A)P(A)}{P(B)}$$

在机器学习的分类任务中，我们通常把$A$替换为类别$y$，把$B$替换为特征向量$X = (x_1, x_2, ..., x_n)$。公式变为：

$$P(y|X) = \frac{P(X|y)P(y)}{P(X)}$$

- **$P(y)$（先验概率 Prior）：**在没看到特征前，某个类别本身存在的概率（例如垃圾邮件在所有邮件中的基础占比）。
    
- **$P(X|y)$（似然概率 Likelihood）：**在已知是某个类别的情况下，这些特征出现的概率。
    
- **$P(y|X)$（后验概率 后验概率）：**我们最终要求解的目标——在看到具体特征后，它是某个类别的概率。
    

### 1.2 为什么叫“朴素 （Naive）”?—— 条件独立性假设

在现实中，特征向量$X$往往包含成千上万个特征（比如文本中的所有单词）。要直接计算联合似然概率$P(x_1, x_2, ..., x_n|y)$几乎是不可能的，因为维度灾难会导致数据极其稀疏。

为了打破这个计算瓶颈，**朴素贝叶斯提出了一个极其强（甚至有点不切实际）的假设：在给定类别$y$的情况下，所有特征之间是相互独立的。**

基于这个假设，联合概率可以拆解为各个特征概率的连乘：

$$P(x_1, x_2, ..., x_n|y) = \prod_{i=1}^{n} P(x_i|y)$$

此时，分类器的判别公式被极大地简化为：

$$\hat{y} = \arg\max_{y} P(y) \prod_{i=1}^{n} P(x_i|y)$$

_(注：分母$P(X)$对所有类别都是一样的，因此在比较大小时可以直接忽略。)_

## 2. 朴素贝叶斯在文本领域的优势与局限

尽管深度学习（如 Transformer/BERT）在今天主导了 NLP，但朴素贝叶斯依然在工业界作为基线模型 （Baseline） 活跃着，这源于它独特的特性。

### 2.1 核心优势

- **计算速度极快：** 训练过程本质上只是对词频进行计数，没有复杂的迭代优化（不需要梯度下降），时间复杂度通常为$O(N \times d)$。
    
- **天然契合高维稀疏数据：** 文本向量化后的矩阵维度极高且极其稀疏。逻辑回归或 SVM 在这种数据上容易过拟合或计算缓慢，而朴素贝叶斯通过概率连乘能够极其高效地处理。
    
- **小样本表现优异：** 即使训练数据量较少，只要先验概率和似然概率的统计具有一定代表性，模型依然能输出合理的预测。
    
- **对缺失/无关特征鲁棒：** 文本中出现未见过的生僻词或无关痛痒的词语，对连乘结果的干扰较小。
    

### 2.2 核心局限性

- **“条件独立”假设在文本中完全不成立：** 语言是具有强烈上下文逻辑的。例如，“机器”和“学习”这两个词同时出现的概率，远大于它们各自概率的乘积。朴素贝叶斯完全丢失了**词序特征**和**词汇关联特征**。
    
- **零概率问题 （Zero-Frequency Problem）：** 如果测试集中出现了一个训练集中从未见过的单词，它的似然概率$P(x_i|y)$将为 0。根据连乘公式，整个文档属于该类别的概率将直接清零。
    
    - _工程解法：_ 引入 **拉普拉斯平滑 （Laplace Smoothing），**在分子和分母上加上一个小常数，确保没有任何特征的概率为绝对的 0。
        
- **概率输出的绝对值不准：** 虽然朴素贝叶斯分类的排序（谁大谁小）是正确的，但由于违背了独立性假设，它输出的最终概率值往往过于极端（比如给出 0.99999 的置信度），不能作为真实良好的概率校准。
