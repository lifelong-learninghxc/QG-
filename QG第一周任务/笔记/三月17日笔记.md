## 🌿 二、版本控制（Git）核心笔记
今天重点理解了代码版本管理与分支协作流程。

### 1. 基础指令（Workflow）
- **git init**：初始化本地仓库
- **git add .**：将修改添加到暂存区
- **git commit -m "message"**：记录一次代码快照
- **git status**：查看当前文件状态

### 2. 分支控制（Branching）
- **git branch <name>**：创建新分支
- **git switch <name>**：切换分支
- **git merge <name>**：合并分支并处理冲突

## 📊 三、数据科学“三大件”实战演练
实战任务：**虚拟咖啡店运营分析**
目标：梳理从数据生成、清洗、统计到可视化的完整流程。

### 1. NumPy（数据原材料）
- 使用 **np.random.randint** 生成模拟销售数据。
- 利用**广播机制**完成销售额计算（销量 × 单价），替代循环，提高效率。

### 2. Pandas（报表管理）
- 将 NumPy 矩阵封装为 **DataFrame**。
- 使用 **df.loc** 进行指定位置数据提取。
- 缺失值处理：用 **np.nan** 模拟缺失，并按业务逻辑进行均值填充。
- 使用 **sum(axis=1)** 快速生成“每日销售总额”列。

### 3. Matplotlib（可视化）
- 绘制折线图展示销售趋势。
- 学会设置图表标题、坐标轴标签和网格。
- 解决中文显示异常问题（配置 SimHei 字体）。

## 💡 今日感悟
- **分块运行**是 Jupyter 的核心优势，可以快速定位问题、降低调试成本。
- 报错（如 NameError）并不可怕，很多时候是变量定义顺序或执行上下文导致。
- 先理解代码按什么顺序执行，再排查错误，效率更高。
